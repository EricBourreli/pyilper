Release Notes for pyILPER 1.6.0

pyILPER 1.6.0 introduces an emulated HP82162A printer as new devices and includes major improvements of the user interface.

New features and enhancements

  • The HP82162A printer is now included as virtual HP-IL device. The device provides high quality output to a PDF file. Add this device with the Virtual HP-IL device configuration in the file menu.
  • Resizing the main window of pyILPER now adjusts the number of lines of the printer, scope and drive directory automatically. This also applies to the size of the plotter viewport. The dimensions of the pyILPER main window is restored at start of the application.
  • Any change of GUI related configuration parameters (font sizes, terminal color scheme, number of columns of the terminal widget) now effects immediately and does not require a restart of the application any more.
  • The terminal, printer and scope now support copying selected text to the system clipboard. The terminal supports pasting text from the system clipboard.
  • Barcodes of HP-41 program and SDATA files can now be created as PDF files from the context menu of the directory listing.
  • Terminal, scope and printers got mouse wheel scrolling support. Note: the focus must be in the output window.
  • A PDF file of the directory listing can now be created from the tool menu of the drive tab.
  • The scope got an option to display the frames as hex codes.

Changes and incompatibilities

  • If a virtual device is re enabled again, its HP-IL address is reset to zero if any HP-IL operation that changed addresses of other devices (AAU, AAD, AEP or AES) happened in the meantime. You must reconfigure the loop (run PILTERM) to access the device again. This prevents duplicate HP-IL addresses of virtual devices which can cause data corruption of LIF media.
  • The PILTERM utility was updated to version 2.1. This version deassigns any DISPLAY IS, PRINTER IS and KEYBOARD IS assignments before issuing a RESTORE IO to avoid possible data corruptions (see the caveat section here. All users are encouraged to update to this version. This prevents assignments of the display, printer or keyboard device to a virtual drive which can cause data corruption of LIF media.
  • For this reason the default pyILPER configurations now puts the virtual drive tabs at the end of the tab menu and therefore at the end of the virtual device chain.
  • The paper size option was moved from the plotter configuration to the pyILPER configuration in the file menu. This parameter now controls the medium size of all PDF output as well as important plotter settings. The default for this parameter is A4. Reconfiguration required if the letter format is used!
  • The way to disable scrolling changed as well. Set the scroll up buffer size to the favored number of lines. Then resize the main window until all lines are displayed. The minimum number of lines of the scroll up buffer is 24.
  • The terminal size configuration parameter was replaced by the terminal width parameter which specifies the number of columns (80 or 120). Reconfiguration required!
  • pyILPER needs the updated version 1.7.7 of the LIFUTILS. Previous installed versions of the LIFUTILS must be updated.

See the Change Log for a full list of changes.
