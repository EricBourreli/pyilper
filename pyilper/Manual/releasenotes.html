Release Notes for pyILPER 1.6.0

pyILPER 1.6.0 introduces an emulated HP82162A printer as new devices and includes major improvements to customize the user interface. Directory listings, printer output and barcodes can be created as high quality PDF files.

New features and enhancements

  • The HP82162A printer is now included as virtual HP-IL device. The device provides high quality output to a PDF file. Add this device with the Virtual HP-IL device configuration in the file menu.
  • Resizing the main window of pyILPER now adjusts the number of lines of the printer, scope and drive directory output as well as the size of the plotter output automatically. The size of the main window is restored at start of the application.
  • The change of GUI related configuration parameters (Font sizes, terminal color scheme, number of columns of the terminal widget) now effects immediately and does not require a restart of the application any more.
  • Barcodes can be created from the context menu in the drive tab for HP-41 program and SDATA files.
  • A PDF file of the directory listing can be created from the tool menu of the drive tab.

Changes and incompatibilities

  • The paper size option was moved from the plotter configuration to the pyILPER configuration in the file menu. This parameter now controls the medium size of all PDF output as well as important plotter settings. The default for this parameter is A4. Reconfiguration required if letter format is used!
  • The way to disable scrolling changed as well. Set the scroll up buffer size to the favored number of lines. Then resize the main window until all lines are displayed. The minimum number of lines of the scroll up buffer is 24.
  • The terminal size configuration parameter was replaced by the terminal width parameter which specifies the number of columns (80 or 120). Reconfiguration required!

See the Change Log for a full list of changes.
