HP2225B Tab

Description

This virtual HP-IL device emulates a HP2225B thermal printer.

HP-IL configuration:

Accessory ID0x23
Default Address5
Device IDHP2225B

This virtual HP-IL device emulates a HP2225B thermal printer. The formatted printer output is sent to a scrollable output window. The buffer size of the printer window is configurable can keep up to 50 pages of graphics or text. The content of the display buffer can be transferred to a high quality PDF file:

  • Push the Clear button to clear the scrollable printer output window. Note: this also resets your printer settings
  • Print a document on the printer. The formatted output is shown in the scrollable printer window
  • Generate a PDF file from the content of the scollable printe window.

Note: Line feeds are not visible in the scrollable display. However they trigger a page break in the PDF file.

Differences from the real device

The emulated device differs from the original device in the following ways:

  • The HP2225B printer bitmap font is emulated with a monospaced outline font. Character sizes are close to those of the original printer.
  • Graphics dots are squares. Therefore graphic lines are less smoothe and less solid than those of the original printer output.
  • Printable output is generated as PDF file. The page size is limited to the dimensions of either the Letter or the A4 format which is specified in the pyILPER system settings. Therefore the printer settings for page length and perforation skip have no effect.
  • The maximum horizontal size of the text area is 10 inches, which is equivalent to 80 lines at 8 lines/inch or 60 lines at 6 lines/inch. The text length can be changed within these limits. A smaller text are is vertically centered in the PDF file. Note: The PDF file is generated only with the endmost text length setting!.
  • The HP-IL printer status bytes always return the condition "ready to receive data" because the error conditions of the real device do not apply to the emulator.
  • Printer controls for selftest or bidirectional printing have no effect.

Issues

  • On macOS stretched characters do not render correctly. This is probably a problem with the Qt version for this operating system. As a work around the expanded font is displayed with unstretched characters but the correct character spacing on macOS.
  • The alternate control mode is not implemented.

Device controls

The HP2225B printer tab has the following controls:

  • Clear: Resets the device in the same way as a HP-IL device clear. The scrollable display is cleared and the emulated printer is put to a default condition.
  • LF: Does a line feed on the printer. If the button is pushed for more than half a second multiple line feeds are generated until the button is released.
  • FF: Does a form feed on the printer.
  • PDF: Generate a PDF file with the content of the printer display buffer. You may choose a filename, a header and the number of columns (1-3).
  • Device enabled checkbox: If checked, the virtual HP-IL printer is available and can be used as HP-IL device. The HP-IL loop must always be reconfigured after activation or deactivation.
  • Log HP2225B checkbox: Activates writing printer output to the file Devicename.log in the pyILPER working directory. Output is always appended to that file. A time stamp in the file marks the start and the end of each logging. The log file is closed automatically on program termination. Note: only printable text is logged to the file.

Tab configuration menu

  • Screen width: You can set the width of the scrollable printer display to 640, 960 or 1280 pixel.
  • Buffer size: Size of the scrollable printer window in pages (approximately).
  • Print color: Select black, red, blue or green color to emulate the color ink cartridges which were available for that device. The color change takes effect immediately for the whole scrollable printer window.
  • Log buffering: enable or disable buffering of the log file. If log buffering is off the log file output is flushed to disk after an end of line was encountered. This enables watching the log file output but may decrease the execution speed of pyILPER considerably.
