Getting started with pyILPER

 pyILPER Manual

Setup pyILPER with the PIL-Box

If you use pyILPER for the first time, please read this document to get pyILPER up and running with the PIL-Box.

To use the file management capabilities of the virtual drives or the HP7470A plotter the latest version of the LIFUTILS must be installed. After installation of the LIFUTILS restart pyILPER and check from the "Utilities" menu if the LIFUTILS version was detected by pyILPER.

Connect the PIL-Box to your computer.

Run "pyILPER configuration" from the "File" menu and configure the tty device of the PIL-Box and the pyILPER working directory.

Open the pyILPER configuration dialog from the file menu. Change the name of the PIL-Box device to the USB serial device the box is connected to. pyILPER tries to figure out the proper device name. If that does not work, configure the device manually and check the Installation instructions for further details.

pyILPER tries to determine the baud rate of the PIL-BOX serial device. If that does not work, set the baud rate manually to the baud rate the PIL-Box is configured to. See the PIL-Box documentation for further details.

pyILPER stores all log files in a working directory. The default of that directory is the users home directory. Change the working directory to an appropriate location.

If the message "connected to PIL-BOX at xxx baud" is displayed in the status bar, pyILPER is ready to run. Now you have to enable each virtual device (check box at the lower left corner of a device tab), because they are disabled by default.

Now initialize the loop with the HP-IL controller.

  • HP-41: switch the calculator OFF and ON.
  • HP-71: issue an RESET HPIL @ RESTORE IO @ CLEAR

See the pyILPER documentation for further details.

Setup the terminal emulator (HP-71 only)

You can use pyILPER as a remote display and keyboard for the HP-71B. See the documentation of the virtual terminal for further information.

Setup pyILPER with virtual HP-IL over TCP/IP

pyILPER supports virtual HP-IL over TCP/IP. You may hook the HP-71B Emulator of Christoph Gießelink with pyILPER.

Change the communication mode to TCP/IP in the pyILPER configuration. Check if the TCP/IP configuration parameters (remote host, remote port, local port) are configured properly. See the documentation of the virtual HP-IL over TCP/IP and the documentation of EMU71 for further details.

The most recent version of EMU71 includes remote keyboard support. Setup the terminal emulator as documented above.
